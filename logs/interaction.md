# 人工交互摘要

1. 用户明确作品参加 LLM Wiki 赛题，要求优化后直接合入 GitHub `main`。
2. 用户指定评测入口为根目录 `INSTRUCTION.md`，平台运行时只使用 OpenCode，并要求说明 GLM-5.2 的外部凭据配置与调用方式。
3. 用户补充正式数据位于 `/app/code/judge-assets/01_01_llm_wiki/`，交付 ZIP 不应包含评测数据。
4. 用户要求固定文件格式与输出合同采用确定性解析；题型归类、模糊安全判断、知识回答和自由批注修复交给子 Agent，不以大量题干正则代替语义判断。
5. 用户提供公开 8 题及参考答案，用于构造脱敏的本地合成夹具和逐字段回归。
6. 用户指定最终文件名及根目录均为 `01_01_硬控AI三秒钟`。

模型凭据仅配置在作品目录之外的 OpenCode 用户配置中；本记录、代码、日志和压缩包均不包含凭据值。
